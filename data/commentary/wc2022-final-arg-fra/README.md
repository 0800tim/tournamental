# WC2022 Final ARG-FRA - VTourn Commentary Pack

Broadcast-density play-by-play commentary for the 2022 FIFA World Cup Final
(Argentina 3-3 France, 4-2 on penalties at Lusail Stadium, 18 December 2022).

This pack drives the renderer's audio overlay. Each line is a discrete TTS
unit (rendered to `Lxxxx.mp3` by ElevenLabs) and is timeline-aligned with
the StatsBomb event stream in
`apps/web/public/data/arfr-stream/fifa-wc-2022-final-arg-fra-2022-12-18.ndjson`.

## Files

| File | Purpose |
| ---- | ------- |
| `manifest.json` | Master index. Every `Lxxxx` line, its `t_ms`, intent, and `duration_ms_est`. Renderer hooks read `cues[]` for goal/sub/whistle moments. |
| `en.md` | English transcript. The primary VTourn narrator output. |
| `es.md` `fr.md` `pt.md` `de.md` `it.md` `zh.md` `ar.md` `ja.md` `ru.md` | Translations. Same `Lxxxx` ids and same `t_ms` so per-line MP3s seek-align across languages. |
| `IMPORT-INTO-ELEVENLABS.md` | Step-by-step ElevenLabs Studio import workflow. |

## Format

Each line in every `*.md` is exactly one timestamped utterance:

```
[L0001 · -03:00 · open] Welcome to Lusail Stadium for the FIFA World Cup Final.
```

- `Lxxxx` - stable id across languages.
- `MM:SS` - game-clock time. Negative for pre-match. Game starts at `00:00`.
- `intent` - semantic category (`open`, `lineup`, `play`, `pass`, `foul`, `shot`,
  `save`, `GOAL`, `tension`, `react`, `sub`, `halftime`, `whistle`, `restart`,
  `tactical`, `history`, `scene`, `shootout`, `tension`, `trophy`, `wrap`).
  TTS prosody can be tuned per-intent (e.g. `GOAL` lines get higher energy).

## Density

```
Total lines:                  ~2965
Pre-match block:              -3:00 to 0:00 (about 56 lines)
Live play coverage:            0:00 to 124:07 (90 + 30 minutes plus stoppage)
Penalty shootout:             124:23 to 127:00 (about 47 lines)
Trophy lift / wrap:           127:00 to 129:00 (about 30 lines)
Average lines per minute:     22.5
Min separation between lines: 200 ms
Max gap between lines:        ~6.5 s (during natural lulls)
```

## Player names policy

Player surnames are kept in their original spelling across all languages
(Messi, Mbappé, Di María, Tchouaméni, Di María). This keeps the audio
consistent across the localised packs and avoids transliteration confusion
in the renderer's nameplate overlay.

Where a name is ambiguous (two players named Martínez) we use:
- "Messi" for Lionel Messi.
- "E. Martínez" for the Argentinian goalkeeper Damián Emiliano Martínez.
- "Lautaro" for Lautaro Martínez.

## Cues for the renderer

`manifest.json` includes a `cues[]` array with renderer hook points:

| Kind | When fired | Use in renderer |
| ---- | ---------- | --------------- |
| `goal` | At each of the six game-time goals | Replay highlight, score-card flash |
| `sub` | At each substitution | Sub overlay, player swap animation |
| `card` | At each yellow card | Card flash, ref animation |
| `pen_attempt` | Per shootout attempt | Frame keeper + striker, freeze pre-kick |
| `whistle` | Half-time, full-time, ET end | Camera cut to wide |
| `win` | Final whistle of the shootout | Trophy lift cinematic |

## ElevenLabs cost estimate

```
Per line:               ~10 spoken words (~55 characters with spaces)
Per language file:      2965 lines x 55 chars = ~163,000 characters
Across 10 languages:    ~1.63 million characters
ElevenLabs Creator:     1M chars / month at USD $99
ElevenLabs Pro:         2M chars / month at USD $330
```

A single full re-render of all 10 language packs costs roughly **USD $200-260** at
Creator-tier rates if rendered in two months, or **one month of Pro** if rendered
in a single sitting. Rendering English first (the master) and reviewing the
manifest before committing budget to translations is the recommended workflow.

Per-line MP3 chunking is the recommended delivery format - the renderer can
preload the next 4-6 lines based on `t_ms` cursor and discard old MP3s, so
total in-flight memory stays under 8 MB even on mobile.

## Translation methodology

The English transcript is the master copy. Translations were generated by
extracting the 822 unique English phrases (most templated phrases recur many
times in different timestamp slots), machine-translating each unique phrase
once with consistent terminology, and remapping back into the timestamp
stream. Quality caveats:

- Player names, score lines, and timing words are kept verbatim in all
  languages so the renderer can pin nameplate overlays to the right moments.
- For broadcast use, native-speaker review is recommended before recording
  with ElevenLabs voice clones. Especially: Arabic word order, Japanese
  particles after Latin player names, Chinese intensifier choice on goal
  bursts.
- See `IMPORT-INTO-ELEVENLABS.md` for the recommended QA loop.

## Generating from scratch

The transcript is regenerated by:

```
cd /tmp/vtourn-build
python3 generate.py    # rebuilds en.md and manifest.json
python3 translate.py   # rebuilds es/fr/pt/de/it/zh/ar/ja/ru .md files
```

`generate.py` is deterministic (seed = match date 20221218); `translate.py`
is cached per-language under `/tmp/vtourn-build/cache/` so re-runs only
translate new phrases.

## Commit hygiene

Both `manifest.json` and the markdown transcripts are committed to the
repo. Audio MP3s are NOT committed; they live on the CDN under
`apps/web/public/audio/commentary/{lang}/Lxxxx.mp3` and are produced by
the ElevenLabs pipeline.

## Source of truth

- Match facts and timings: StatsBomb open data, match id 3869685.
- Pre-match scene-setting: composed from the public broadcast record
  (lineups, attendance, kit colours, Emir's bisht, Mbappé's hat-trick).
- Narrator voice: VTourn original, paraphrased into a single consistent
  play-by-play persona. No third-party broadcaster transcripts were
  reproduced.
