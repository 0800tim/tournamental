## Format

When a credential lands, paste it into the relevant `.env` (or the deployment secret store later) under the named variable. **Never commit `.env` to git.** Only `.env.example` is checked in. The orchestrator records "credential received, wired" in `tasks/done/` when each item closes.

## Process

When a credential lands, the orchestrator:

1. Adds it to `.env` and verifies the wiring with a smoke test.
2. Updates the relevant `task` file in `tasks/done/` with a one-line outcome.
3. Records the date received in this doc next to the row (e.g. add a "received: 2026-05-12" line if helpful).
4. Notifies any waiting builder agent that the integration is unblocked.

If a credential is exposed (committed to git, leaked in a screenshot, etc.):

1. **Rotate it immediately** at the provider.
2. Force-push history rewrite is *not* a fix — assume the secret is burned. Rotate.
3. Document the incident in `sessions/<date>_security-incident_<slug>.md`.
