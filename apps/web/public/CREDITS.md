# Asset Credits

This file documents the licence and provenance of every binary asset
shipped from `apps/web/public/`. Per
[`docs/07-avatars-and-assets.md`](../../docs/07-avatars-and-assets.md)
every asset must be CC0, a CC-BY-compatible Creative Commons licence,
or self-authored. No copyrighted club crests, sponsor logos, real-player
photos, or licensed fonts are bundled here. Player face images are
fetched at runtime from URLs in `data/wc2022-final-players.csv` — the
runtime referencer is the operator, the renderer is not the publisher.

## Models

### `models/body.glb`

- **Authored by**: Tournamental (this repo). Programmatically generated by
  [`packages/avatar/scripts/build-assets.mjs`](../../packages/avatar/scripts/build-assets.mjs).
- **Licence**: CC0 1.0 (Public Domain Dedication). Self-authored. Verify
  by re-running the build script.
- **Description**: Low-poly humanoid (~800 tris, well under the 3K
  budget in doc 07), Mixamo-named bone hierarchy at T-pose. Sub-meshes
  named `torso`, `shorts`, `socks`, `head_billboard` so the renderer
  can paint each region independently at runtime.

## Animations

### `animations/*.glb` (15 files)

- **Authored by**: Tournamental (this repo). Programmatically generated by
  [`packages/avatar/scripts/build-assets.mjs`](../../packages/avatar/scripts/build-assets.mjs).
- **Licence**: CC0 1.0.
- **v0.1 status**: `idle.glb`, `run.glb`, and `kick.glb` are unique seed
  loops authored on the body's Mixamo skeleton. The other 12 files
  (`walk`, `sprint`, `pass`, `header`, `shoot`, `tackle`, `fall`,
  `celebrate`, `throw`, `catch`, `dribble`, `jump`) ship as idle-clip
  stubs so the renderer's animation loader contract works for all 15
  spec `AnimTag` values without throwing. Mixamo retargets for the
  remaining 12 are tracked in `IDEAS.md` for v0.2.

## Player face images

### `data/wc2022-final-players.csv`

- **Source**: Wikidata `image` claim (P18) → Wikimedia Commons. Each
  thumbnail URL uses `Special:FilePath` so the URL stays canonical even
  if a file is renamed.
- **Per-file licence**: Recorded in the `attribution` column of the CSV.
  Most rows are CC-BY-3.0 / CC-BY-SA-3.0 / CC-BY-4.0 / CC-BY-SA-4.0
  uploads from Wikimedia Commons. Each row carries the original
  uploader credit and the SDC-declared licence as of build time.
- **Operator obligations**: When the renderer uses these images, it must
  display the licence and a link back to the Commons file page. Per
  doc 07, face images are referenced by URL, not bundled with the
  renderer — the operator (i.e. the StatsBomb-replay producer that
  emits MatchInit) is responsible for that link in the HUD.
- **Refresh**: re-run `node packages/avatar/scripts/build-players-csv.mjs`
  (requires network) to refetch P18 values from Wikidata. The script
  caches nothing, so the resulting URLs always reflect the live state
  of Wikidata.

## Fonts

The jersey-number text uses the system font stack
`Inter, Arial, sans-serif`. **No font files** are bundled. If the
renderer's host page wants Inter specifically, pull it from the system
or from Google Fonts at runtime under the [SIL Open Font Licence
1.1](https://fonts.google.com/specimen/Inter/license) — but doc 07 says
"do not check in licensed fonts" and we follow that.

## Verification

To re-verify all attributions, run:

```bash
# Rebuild the body + animation pack from source (CC0)
node packages/avatar/scripts/build-assets.mjs

# Refetch player photo URLs + licences from Wikidata + Commons
node packages/avatar/scripts/build-players-csv.mjs
```

The first command writes to `apps/web/public/models/` and
`apps/web/public/animations/`; the second writes to
`data/wc2022-final-players.csv`. Both are idempotent.
