## Replay (`/match/[id]`)

- **Renderer fidelity overhaul** (#107). Tim watched the AR-FR demo
  on his phone and reported the field jerks back and forth. Diagnosed
  four root causes: synthetic-stream pacing was collapsing
  interpolation (multiple state frames stamping the same wall-clock
  instant, so `prevWallMs === currWallMs` and `alphaForNow` saturated
  at 1, snapping to `curr` every tick), camera writes were unsmoothed,
  `useFrame` deltas were unclamped on stalls, and the pitch grass
  texture was shimmering at glancing angles. Eight fixes shipped:
  match-time `StateFrameBuffer` (linear interp on position, slerp on
  yaw, Catmull-Rom on the ball), `DampedCameraDriver` with
  frame-rate-independent damping (λ=5 position, 4 lookAt, 6 FOV),
  `delta = Math.min(deltaRaw, 1/30)` everywhere, 2048×1024 pitch
  canvas with explicit mipmaps, throttled crowd material updates
  (4 Hz with hysteresis), throttled GoalNet sway (5 Hz), and per-profile
  DPR caps `[1.0, 1.5, 1.75]` for low/medium/high. 17 buffer tests
  + 6 driver tests added.
- The full diagnosis and fix list is in
  [docs/43-renderer-fidelity-overhaul.md](../../docs/43-renderer-fidelity-overhaul.md).

## Bracket (`/world-cup-2026`) and team pages

- **Per-match pick popup** (#108). The bracket page is the right
  surface for the "lock-it-all-in" cascade, but team pages, match
  preview pages and (later) social cards present an individual fixture.
  `MatchPickPopup` ships in three presentations — `sheet`,
  `modal`, `inline` — drag-to-close on touch, Escape on keyboard,
  same kit-coloured selection ring as the bracket row. Backed by three
  new endpoints on `apps/game`: `PUT /v1/picks/:userId/:matchId`,
  `GET /v1/picks/:userId/:matchId?tournament_id=...`,
  `DELETE /v1/picks/:userId/:matchId?tournament_id=...`. All
  `private, no-store`. Per-match writes are read-modify-write on the
  same `brackets.payload_json` row the bulk submit handler touches,
  so a user can pick game-by-game and then later open the bulk
  builder to find every pick already populated. Spec at
  [docs/45-per-match-pick-popup-and-api.md](../../docs/45-per-match-pick-popup-and-api.md).
- **Mobile-first overlay/sheet system + deep-linkable cards** (#109).
  Tim's directive: the bracket should feel card-style on mobile - tap
  a team, the team card overlays the bracket; tap close, drag down,
  or press browser back to return where you were. Custom `OverlayRouter`
  ships with `?overlay=team,match&code=NZL&id=55` URL encoding,
  underlying-route preservation (page route does not change when an
  overlay opens), and `popstate`-driven stack unwind. 30+ unit tests
  in `overlay-router.test.tsx`. The bottom sheet on mobile reuses the
  same `Sheet` chrome as the pick popup. Sample deep link
  `/team/NZL?pick=55` opens the popup pre-loaded. Spec at
  [docs/44-overlay-router-and-mobile-overlays.md](../../docs/44-overlay-router-and-mobile-overlays.md).
- **MatchPredictionRow enrichment** (#103). Per the PR #79 UX
  dossier: `FormDots` (W/D/L last-five), `HeadToHeadPill`, and a
  kit-coloured selection ring around the picked team's flag.
- **Mobile bracket gestures** (#94). Pinch-zoom on the knockout
  grid, sticky group headers, haptic feedback on each pick (vibrate

## Backend services (4 new apps)

- **`apps/wc2026-data`** (#101) — Live-data scraper for the actual
  2026 matches. Provider-pluggable: SportRadar, API-Football, and a
  mock provider for dev. Scheduled fetcher with Redis-backed cursor
  state, settlement bridge into `apps/game`. Real keys are env-flagged;
  mock is the default until Tim acquires a paid feed. Spec at
  [docs/42-wc2026-live-data.md](../../docs/42-wc2026-live-data.md).
- **`apps/dm-poll-forwarder`** (#102) — Polling worker for Reddit,
  Mastodon and Signal DM-OTP. The 16-channel DM-OTP service that
  shipped yesterday left these three behind because none of them
  push webhooks the way Telegram and Discord do; this worker
  inverts the relationship — it polls each platform's "unread DMs"
  endpoint on a per-platform cadence and forwards new messages
  back into `apps/dm-otp` via the existing webhook contract.
  Cursors persisted to JSONL so a worker restart does not re-flood
  the OTP service.
- **`apps/drips-bridge`** (#100) — Drips Network contributor
  revenue-split scaffold. Mock Drips client today; the real client
  is gated on contract audit. Reads contributor activity (commits,
  reviews, merges) and computes a normalised weight per contributor
  per epoch. Hooks for the on-chain split call exist; signed-tx
  output is stubbed. Spec at
  [docs/40-drips-network-integration.md](../../docs/40-drips-network-integration.md).
- **`apps/native`** (#99) — Capacitor iOS + Android shell wrapping
  `apps/web`. Push notifications via Capacitor Push, haptics via
  Capacitor Haptics (the same plugin the bracket gestures wire),
  share via Capacitor Share. `npm run android` and `npm run ios`
  scripts boot the simulators. App store submission is a follow-up
  once the icons + screenshots land.

## Security & CI/CD

- **CI/CD with staging/prod slots** (#110). Sdeal's `publish.sh`
  generalised into a typed, tested, monorepo-aware library at
  `infra/deploy/`. Per-app entrypoints, GH Actions workflows for
  `build-and-deploy`. Same-host blue-green slot swap with
  `pm2 reload` for ~2-3 second perceived downtime, smoke test on
  the staging port before swap, persistent `.next/cache` between
  builds for fast rebuilds, parallel build matrix across changed
  apps only. Cheap rollback via the previous slot. Architecture
  in [docs/47-cicd-pipeline.md](../../docs/47-cicd-pipeline.md).
- **PR-triage + security-watchdog (open, #111)**.
  `apps/pr-triage-bot` scores incoming PRs 0-100 deterministically
  from path classifiers + file-type + size, posts a structured
  triage comment, applies labels, routes review requests to
  CODEOWNERS for yellow/red verdicts. `apps/security-watchdog`
  runs scheduled scans (daily deps, weekly full-repo) with
  severity-routed alerts via pluggable sinks (Slack, Discord,
  Telegram, Aiva-SMS, email) and an ack/resolve audit log.
  Defence-in-depth workflows: gitleaks, license audit,
  network-allowlist diff, prompt-injection canary, secret-scope
  check. New `CONTRIBUTING.md` and `SECURITY.md`. CODEOWNERS
  gates `apps/auth-sms`, `apps/identity`, `apps/dm-otp`,
  `apps/vstamp`, `apps/drips-bridge`, `.github/`, and
  `packages/spec/` to @0800tim. Threat model documents PR content
  as untrusted data; the prompt-injection canary only flags,
  never auto-actions.
