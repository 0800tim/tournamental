## Replay (`/match/[id]`)

- Single-source clock fix in `packages/spec-client/src/manifest.ts` —
  the driver was emitting two competing time values causing visible
  HUD jitter; now `controller.getTime()` is the only authoritative
  source. Cursor reset moved to a separate `subscribeSeek` channel so
  natural advance no longer drops events.
- `MatchStatsHUD` broadcast overlay: scoreboard, scorers ticker
  (chronological with side-attribution), stats panel

  56 new tests.

## Bracket (`/world-cup-2026`)

- Third-place playoff stage (Bronze match) added end-to-end through
  `StageId`, fixtures, cascade, and UI.
- Per-match flag pick UX: large circular flags + 3-letter code + DRAW
  pill. Inline W/D/L percentages under each flag, fed by a single bulk
  `/api/odds/snapshot` fetch instead of 72 individual calls.
- ⚡ Auto-pick now cascades all 104 picks in one click. Walks
  R32 → R16 → QF → SF → 3rd-place → Final, re-running the cascade
  between rounds so newly-resolved slots populate before we pick.
  Default group tiebreakers applied (FIFA-rank ranking, user can
  override).
- Auto-pick confirmation modal — explains the action and warns
  existing picks will be overwritten.
- `oddsAtLock` provenance — every prediction now carries the W/D/L
  snapshot at the moment of lock-in, so the scoring engine can later
  reward early picks at higher odds.
- Append-only history ledger at
  `vtorn:bracket:history:v1:<tournament>:<user>` localStorage key.
  Every change (pick, score edit, tiebreaker, auto-pick run) writes a
  `HistoryEntry` with prev outcome, new outcome, odds snapshot, ISO
  timestamp.
- Kickoff lockout — client and server. The row flips into a locked
  state at `now >= kickoff`, all buttons go disabled, banner reads
  "Sorry — this match has already started. You can't change it now."
  Server-side enforcement in `apps/game` rejects late submissions
  with `match_already_started` per match.

## Backend services

- `apps/social-publisher` (:3382) — 9 platform adapters: TikTok,
  Instagram Reels, YouTube Shorts, X, Threads, Telegram, Discord,
  Reddit, plus the Aiva-WhatsApp adapter that fans clips into
  configured WA groups. Per-tournament policy table, JSONL audit log,
  per-platform rate limits, metrics-pull skeletons.
- `apps/crm-bridge` (:3395) — GoHighLevel sync. Real HTTP client
  behind `CRM_BACKEND=real`. Mock/JSONL by default.
  `POST /v1/events/{user_signup,prediction_locked,syndicate_joined,
  bracket_shared,match_settled}` upsert contacts + custom fields +
  tags. `GET /v1/customer/:userId` returns the aggregate. Retry +
  backoff on 429/5xx, dead-letter to `data/ghl-failed.jsonl`,
  super-admin `POST /v1/admin/replay-failed` to walk failures.
- `apps/identity` (:3392) — OAuth link adapter stubs for Google,
  Apple, Telegram, X, Discord, Phone. Humanness Score 0-100 with
  explicit factor weights (provider stack, link freshness, behaviour
  consistency, device fingerprint, captcha, bot signals).
- `apps/push-notifications` (:3398) — Web Push + Telegram + SMS +
  WhatsApp fan-out with `auto`-prefer policy (WA over SMS when both
  linked). Kickoff scheduler scans fixtures and arms timers for
  `kickoff − 30min` and `kickoff − 5min`. Subscription store with
  consent-flag enforcement.
- `apps/dm-otp` (:3331) — User-initiated DM-OTP login across 16
  channels: Telegram, WhatsApp, Messenger, Instagram, Discord, X,
  Reddit, Threads, Slack, Mastodon, LINE, Viber, Microsoft Teams,
  LinkedIn (partner-gated), Signal (self-hosted), Email magic-link.
  User types "log in" to any of our official accounts; we reply with
  a 6-digit code; they verify on the website. Side-steps the Meta
  24-hour rule because the user initiates.
