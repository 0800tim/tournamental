# Capacitor native iOS+Android shell

- **Agent**: native-builder
- **Branch**: `feat/capacitor-native-shell`
- **Status**: ready-for-review

## Plan

Wrap the production `apps/web` deployment in a Capacitor 6 shell so we
can ship to App Store + Play Store without rebuilding the renderer in
SwiftUI / Compose. Wire native push (APNs / FCM) through to the existing
`apps/push-notifications` service via a new `/v1/subscribe/native`
endpoint, and add native haptics + share sheet hooks so the wrapped
WebView feels native on the bracket page and the lock page.

## What shipped

### `apps/native/` (new)

- `capacitor.config.ts` — appId `com.vtourn.app`, appName `VTourn`, the
  WebView's `server.url` defaults to `https://vtourn.com` and is
  overridable per-build via `VTORN_WEB_URL`. Cleartext is auto-enabled
  for HTTP (dev only).
- `src/native-bridge.ts` — the JS surface the wrapped web app talks to
  on native. Wires push permission + register + token POST, exposes
  `tapFeedback()`, `shareContent()`, and a `startNativeShell()` boot.
  Persists a stable `userId` in `@capacitor/preferences`.
- `scripts/generate-icons.ts` — placeholder V mark on accent-blue
  background, written as 1024 / 512 / 192 PNGs (via `@resvg/resvg-js`)
  to `resources/icon/`. Also writes `resources/splash/splash-2732.png`.
- `scripts/build-web-bundle.ts` — minimal offline `www/index.html` shell
  shown only if `server.url` is unset (e.g. for sandboxed app-store
  review builds).
- `ios/` and `android/` folders generated by `cap add` and committed.
  `Pods/`, `build/`, `.gradle/`, and signing material are gitignored.
- README covers build commands, sim/emu testing, env-vars, what's
  stubbed, and the developer-account setup Tim needs to do tomorrow.

### `apps/push-notifications/` (extended)

- New `POST /v1/subscribe/native` route taking
  `{ userId, consent: true, platform: 'ios'|'android', token }`.
  Returns 201 on success, 400 on bad input. Token is masked in the
  audit log (first 4 + last 4 chars only) — full device tokens never
  hit disk.
- `SubscriptionStore.upsertNative()` and `getNative()` added; the
  `Channel` and `AuditChannel` unions extended with `'native'`.
- 6 new server tests + 1 store test (45 tests passing total).

### `apps/web/` (light wiring)

- `lib/native.ts` — small shim that dynamic-imports the Capacitor
  packages **only** when `window.Capacitor.isNativePlatform()` is
  truthy. Web bundle stays unaffected for non-native users.
- `components/NativeShellBoot.tsx` — mounted from the root layout;
  no-op on web, calls `startNativeShell()` on native.
- `components/bracket/MatchPredictionRow.tsx` — pick-feedback now
  fires `tapFeedback("light")` on every outcome change. Falls back
  to `navigator.vibrate(10)` on plain web.
- `components/bracket/LockSummary.tsx` — new "Share my bracket"
  button that uses the native share sheet (Capacitor) on device,
  Web Share API on supported browsers, and clipboard fallback
  otherwise. Shares the canonical `/world-cup-2026/share/<id>` URL
  with handle + winner query params.
